Another digested hex view for human's quick recognition.

goal

  • To help human's quick recognition of familiar things from different or strange others when encountering very frequently.
  • One situation is to use SHA-256 for user's ID or other's ID while people's name or nickname can be same or be duplicated evilly.
  • It's not to retrieve accurate info back again, but to find familiar one easily by color and type if anything can be changed to digital resource.

idea, how to create

  1. The 24bit/3byte RGB color is minimized to 8 colors. RGB = 0|1, 0|1, 0|1
  2. The first and last HEX are used for deciding pattern.
    All are 7 types : normal, top, middle, bottom, half, diagonal, diagonal2.
  3. The second HEX from the beginnning decides if it's rotated or not.
  4. The second HEX from the end decides if it's round one or not.
  5. The rest hex will be printed out together if those are not used to make.
